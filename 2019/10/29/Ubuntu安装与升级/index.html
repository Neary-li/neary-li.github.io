Ubuntu系统的升级、node环境安装和wireshark安装教程

Author: Neary Li Date: October 29, 2019  20:37:28 Category: Ubuntu

本章主要内容包括

  • Ubuntu系统的升级
  • node环境安装
  • wireshark安装教程

Ubuntu系统的升级

查看系统信息

  1. 首先进入Ubuntu系统，并鼠标右键点击打开终端，当然也可以使用快捷键Ctrl+Alt+t打开终端。

  2. 输入命令：cat /proc/version查看版本信息。

    • Linux内核版本号：Linux version 4.15.0-43-generic (buildd@lgw01-amd64-001)。

    • gcc编译器版本号：gcc version 7.3.0。

    • Ubuntu版本号：Ubuntu 7.3.0-16ubuntu3。

  3. 输入命令：uname -a。
    显示Linux的内核版本和系统是多少位的：X86_64表示是64位Linux系统。

  4. 输入命令：lsb_release -a。

    • Distributor ID代表类别是Ubuntu。

    • Description表示为18年1月份发布的稳定版本，LTS是Long Term。

    • Codename表示Ubuntu的代号名称。

升级系统

  1. 更新资源update。
    输入命令：sudo apt-get update

  2. 对软件进行升级。
    输入命令：sudo apt-get upgrade

    输入Y，进入下一步。

  3. 检测是否有新版本。
    输入命令：sudo update-manager -c -d

  4. 出现软件更新器界面，有可更新的软件，点击“立即安装”即可。

  5. 安装完成之后立即重启计算机。

  6. 重新输入cat /proc/version查看系统信息。

Ubuntu系统上Node.js和npm安装和更新

首先查看系统上是否安装过node.js和npm。分别输入命令：node -v和npm -v，如果有显示的版本信息，则可以直接查看本文档的升级部分，如果没有显示版本信息，则说明你没有安装，则可直接查看安装部分。

安装教程

  1. 安装node.js。输入命令：sudo apt-get install nodejs-legacy。

  2. 安装npm。输入命令：sudo apt-get install npm。

因为Ubuntu使用apt-get安装时，选用的默认版本都是低版本，所以我们需要对它进行升级。

升级教程

  1. 安装用于安装nodejs的模块n。
    输入命令：sudo apt-get install -g n。

  2. 通过n模块安装指定的nodejs。
    sudo n latest       // 表示最新版node.js
    sudo n stable       // 表示官方稳定版（推荐）
    sudo n lts         // 表示最新稳定版

  3. 升级npm为最新版。
    输入命令：sudo npm install npm@latest -g。

  4. 完成升级后查看版本。
    分别输入命令：node -v和npm -v。

*PS： 在升级node.js的过程中可能会遇到运行sudo n stable时该版本的node.js链接不上无法通过字符匹配的问题和npm error等。

解决办法1： 可以cd /usr/lacal/bin，然后输入ll进行查看，发现node的连接指向自己，然后运行sudo rm node。

解决办法2： 将node,js和npm卸载，重新再安装。
卸载命令：sudo apt-get remove nodejs

安装wireshark

Wireshark是一个网络抓包工具，可以抓取tcp/http/udp/smpf等协议的数据。本教程是基于Linux环境下安装的。

  1. 安装安装包。
    命令：dpkg -l |grep wrieshark          //检查系统是否有这个包
    命令：sudo apt-get-repository ppa:wireshark-dev/stable     //获取wireshark仓库
  2. 如果没有，准备开始安装。
    命令：sudo apt-get update
    命令：sudo apt-get install wireshark
  3. 设置普通用户也可以使用wireshark。
    命令：sudo dpkg-reconfigure wireshark-common
